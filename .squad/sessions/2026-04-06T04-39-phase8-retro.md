## What Worked

1. **Parallel agent waves** — Running 3-4 agents simultaneously compressed a full validation cycle into ~45 minutes. Wave 1 found issues; Wave 2 reviewed and documented.
2. **Live integration testing** — Bender's actual REST/WebSocket tests with Copilot proved the auth pipeline, model resolution, and streaming all work end-to-end. This caught real issues (missing 404/400 validation) that unit tests missed.
3. **Cross-agent artifact flow** — Hermes's tests, Fry's WebUI fixes, and Bender's gateway fixes all landed cleanly. The one conflict (WebSocket DI registration) was caught immediately by the test suite.
4. **Pre-commit hook** — The build+test pre-commit hook caught the DI regression before it could compound. Worth the 4-minute commit times.

## What Didn't Work

1. **WebSocket DI regression** — Bender's well-intentioned change to use factory-delegate registration broke 10 tests on .NET 10. Root cause: `.NET 10's TryAddEnumerable` enforces distinguishable implementation types for factory delegates. The coordinator had to intervene to fix this.
2. **Long-running agent (Bender, 33 min)** — Live integration testing with real API calls + pre-commit hook delays made Bender's run extremely long. Consider: (a) skip pre-commit for integration-test branches, (b) use `--no-verify` for intermediate commits during testing.
3. **Scribe commit timeout** — The pre-commit hook occasionally causes Scribe's commit step to appear stuck. Not a real failure — just slow.

## What to Adjust

1. **Add `--no-verify` option for agent commits during testing** — The pre-commit hook running the full CodingAgent test suite (3.5 min) on every commit slows agent work significantly. Consider a lighter hook for `.squad/` changes.
2. **Sequence DI-touching changes before integration tests** — The DI regression would have been caught earlier if we'd built+tested before Bender started live testing.
3. **Consider integration test markers** — Tests that require a running gateway should be in a separate test category (e.g., `[Trait("Category", "Integration")]`) so they can be run separately from unit tests.
