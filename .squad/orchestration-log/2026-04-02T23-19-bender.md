- Impact: Intermittent build failures, corrupted assemblies

- **Old approach:** `dotnet pack --no-build` with parallel publish (unsafe)
- **New approach:** Sequential `--no-restore` builds with `/p:UseSharedCompilation=false`
- Enforces per-project compilation isolation

- Removed parallel pack execution
- Added shared compilation disable flag to all build commands
- Maintains build speed: project builds still run efficiently (GCC-like incremental compilation)
- Serializes only high-conflict stages
